Overview

The Eternal Guest is a narrative-driven psychological horror game focused on environmental storytelling, player-driven pacing, and systemic tension. I worked as the primary gameplay programmer, owning interaction systems, progression logic, and core architectural decisions.

My Responsibilities

  • Designed and implemented core interaction and examination systems
  • Built event-driven gameplay systems to support narrative pacing
  • Developed save/load and progression tracking systems
  • Collaborated closely with design and narrative to support iteration
  • Maintained clean system boundaries to avoid narrative hard-coupling

Key Gameplay Systems

Interaction & Examination System

Player interactions were implemented using a component-driven architecture that allowed designers to author interactive objects without custom code, supporting rapid iteration and content expansion.

Event-Driven Narrative Triggers

Narrative progression relied on decoupled event triggers and state flags rather than linear scripting, allowing scenes to react dynamically to player behavior and exploration order.

Save / Load & State Persistence

Implemented a lightweight serialization system to persist world state, player progress, and narrative flags across sessions without introducing brittle dependencies between systems.

Technical Challenges & Solutions

  • Prevented narrative logic from becoming tightly coupled to level scripts by centralizing state evaluation and progression rules.
  • Balanced flexibility and control in the interaction system to avoid designer-created edge cases that could break progression.
  • Optimized scene loading and memory usage to reduce hitching during exploration-heavy gameplay.
